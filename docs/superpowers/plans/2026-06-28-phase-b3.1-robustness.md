# Phase B3.1 — Real-Site Robustness Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the agent reliable on real sites by closing the three gaps the live YouTube run exposed: **funnel quality** (`OcclusionCuller` + `WrapperCollapser` — drop covered/nested-wrapper elements so clicks land on real targets), the **missing action tools** (`PressKey`/`Clear`/`SelectOption`/`NewTab`/`SwitchTab`/`CloseTab`), and **vision** (feed the already-captured screenshot to the model, config-gated).

**Architecture:** Three mostly-independent groups on top of B3. ① Two new pure-Python funnel stages, fed by an `occluded` flag the Extract JS computes via `document.elementFromPoint`, wired into `run_funnel`. ② New Pydantic tool specs + `ToolDispatcher` name mappings + `LocalCDPSession` handlers (trusted `page.keyboard`/`page.mouse`/tab management; `select_option` via set-value+dispatch-change per spec §5). ③ A `use_vision` flag that makes the `observe` node attach the screenshot as a multimodal image block.

**Tech Stack:** Playwright (async), langchain-core multimodal messages, Pydantic v2, pytest/pytest-asyncio. (All already installed.)

## Global Constraints

- Work in `backend/`; tests `cd backend && uv run pytest` (use `-m "not browser"` for the fast loop; `browser`-marked tests launch real Chromium).
- **Zero contract/graph-shape changes**: new tools are added to the existing `TOOL_SPECS` + `ToolDispatcher` + `LocalCDPSession` dispatch; `Observation`/`ActionCall`/`BrowserSession` signatures unchanged. Indices still rebuilt every `observe()`; coords still only in the hidden `index_map`.
- **Trusted input**: `PressKey`/`Clear` use `page.keyboard`/`page.mouse` (CDP-backed). `SelectOption` uses a JS `value`-set + `change` dispatch (per spec §5 — native `<select>` dropdowns can't be coordinate-clicked reliably; this is the documented exception).
- **`act()` stays fail-closed** — new dispatch branches must not raise out of `_dispatch`; the broad guard in `act()` already converts errors to `ActionResult(success=False, errorCode="ACTION_ERROR")`, but new branches should still validate args and return graceful failures where natural.
- **Honest scope note:** `OcclusionCuller` fixes *statically* covered elements (modals, fixed overlays). Hover-*triggered* overlays (YouTube's "now playing" preview that appears on mouse-move) are NOT caught at observe-time and remain hard — vision + the model's own retry mitigate those. `WrapperCollapser` reduces nested-wrapper noise (the other half of the YouTube click problem).
- **Vision is OFF by default** (`use_vision=False`) — it ~doubles input tokens. Existing fast tests/e2e stay text-only.
- Reuse everything from A/B1/B2/B3; do not redefine contract types.

```python
from __future__ import annotations

from app.observation.raw import RawElement

_REAL_TAGS = ("a", "button", "input", "select", "textarea", "summary")


class WrapperCollapser:
    """Collapse near-identical-bounds wrapper chains to one representative element."""

    def __init__(self, tol: float = 4.0) -> None:
        self._tol = tol

    def _key(self, e: RawElement) -> tuple:
        t = self._tol
        return (round(e.x / t), round(e.y / t), round(e.width / t), round(e.height / t))

    def apply(self, raw: list[RawElement]) -> list[RawElement]:
        groups: dict[tuple, list[RawElement]] = {}
        order: list[tuple] = []
        for e in raw:
            k = self._key(e)
            if k not in groups:
                groups[k] = []
                order.append(k)
            groups[k].append(e)

        out: list[RawElement] = []
        for k in order:
            members = groups[k]
            real = [m for m in members if m.tag in _REAL_TAGS]
            best = real[0] if real else members[0]
            # prefer a representative that actually has a name, if present
            named = [m for m in (real or members) if m.name]
            out.append(named[0] if named else best)
        return out
```

Module-level constant:

```python
_SELECT_JS = """
([cx, cy, value]) => {
  const el = document.elementFromPoint(cx, cy);
  const sel = el && (el.tagName === 'SELECT' ? el : el.closest('select'));
  if (!sel) return false;
  let opt = [...sel.options].find(o => o.value === value || o.text === value);
  if (!opt) return false;
```

```python
from __future__ import annotations

import base64

from langchain_core.messages import HumanMessage

from app.agent.format import format_observation
from app.agent.state import AgentState
from app.browser.base import BrowserSession
from app.events.emitter import EventEmitter
from app.telemetry.records import StepRecord


def build_observe_node(session: BrowserSession, emitter: EventEmitter, *, use_vision: bool = False):
    async def observe(state: AgentState) -> dict:
        obs = await session.observe()
        await emitter.emit_observation(obs.url, len(obs.elements))
        text = format_observation(obs)
        shot = getattr(session, "latest_screenshot", None)
        if use_vision and shot:
            b64 = base64.b64encode(shot).decode()
            content = [
                {"type": "text", "text": text},
                {"type": "image_url", "image_url": {"url": f"data:image/png;base64,{b64}"}},
            ]
        else:
            content = text
        return {
            "observation": obs,
            "messages": [HumanMessage(content=content)],
            "history": [StepRecord(step=state.step, node="observe")],
        }

    return observe
```
